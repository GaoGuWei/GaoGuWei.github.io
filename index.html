用Hexo搭建一个静态博客框架

hh,欢迎点进来。如果你能找到这篇博客，说明你已经成功了一半。毕竟，我在知道要写博客，到真正动手写博客，就花了1/2的时间。

好了，话不多说，先让我们来看一段视频。

视频链接

如果你有一定的编程基础，相信根据这个视频你就已经能搭建出一个自己的博客框架啦。如果你已经搭出了框架，可以跳过下面的笔记文字。

下面，我就根据这个视频来做一个笔记吧。我用的是Windows10系统，有一些地方和视频会有不同。

首先，你需要的工具有：Node.js、Hexo、github账户
Nodejs下载链接
GitHub网站链接

Step 1：在终端中，用npm安装cnpm镜像源（因为谷歌安装插件是从国外服务器下载，受网络影响大，可能出现异常，所以我们用淘宝的。）。接着用cnpm安装hexo软件，安装git插件。

在常用软件中搜索 cmd 即可打开终端（Mac系统是item）【大概长这样：

安装cnpm镜像源代码：
npm install -g cnpm –registry-https://registry.npm.taobao.org

安装hexo代码：
cnpm install -g hexo-cli

安装git插件代码：
cnpm install –save hexo-deployer-git

Step 2：创建一个文件夹。把所有的编辑内容，都放在这一个文件夹里，如果错了，删除该文件夹就好了。我建在了D盘里。打开D盘blog文件夹，用hexo初始化一个blog。

创建文件夹代码：
mkdir blog

初始化hexo代码：
hexo init

Step 3: hexo里的基本操作。本地查看博客，可以打开任意浏览器，输入：localhost:4000

写一篇博文的代码：
hexo n “My name is CowryGao”

在本地启动博客的代码：
hexo s
【退出ctrl+c】

生成博文的代码：
hexo g

部署博客到远端的代码：
hexo d

清理本地代码：
hexo clean

Step 4: 将本地博客部署到GitHub上。如果你还没有GitHub，戳下面链接，创建一个吧。
https://github.com
在GitHub里创建一个仓库，仓库名就是以后访问你的博客时输入的地址。

修改 _config.yml 文件，我是用notepad++打开的，当然直接用notepad记事本也可以。代码照着写就行，记得每个冒号后加空格。

命令行执行 hexo d 命令，实现部署。记得把你的账户设置为公开。不然会报错。

至于皮肤主题的更改，文章的书写，图片及视频的插入。就放到下一个篇叭。
